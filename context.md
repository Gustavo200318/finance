Crie o design completo de um app financeiro pessoal avançado.

Contexto:
Estou construindo um app de controle financeiro pessoal para pessoas que ganham relativamente bem, mas perderam o controle financeiro, estão endividadas ou querem enxergar melhor para onde o dinheiro está indo.

O app deve ter uma pegada de fintech premium, moderna, jovem e muito clara. Não quero algo genérico. Quero um design com personalidade, com cara de produto real, bonito o suficiente para parecer uma startup financeira séria.

Objetivo do app:
Ajudar o usuário a:
- Entender para onde o dinheiro está indo
- Controlar receitas e despesas
- Cadastrar contas bancárias e cartões
- Acompanhar dívidas
- Ver uma DRE pessoal mensal
- Fazer previsões financeiras
- Importar CSVs futuramente
- Receber insights inteligentes sobre gastos
- Criar metas para sair do negativo e organizar a vida financeira

Nome provisório do app:
Clareza Financeira

Estilo visual:
- Premium
- Minimalista
- Moderno
- Jovem
- Confiável
- Sofisticado
- Inspirado em fintechs, private banking e dashboards SaaS
- Visual escuro elegante, mas com possibilidade de versão clara futuramente

Paleta sugerida:
- Fundo principal: azul-marinho bem escuro ou quase preto
- Cards: cinza grafite / navy mais claro
- Destaques positivos: verde elegante
- Destaques financeiros/premium: dourado suave
- Alertas/dívidas: vermelho sofisticado
- Textos: branco, cinza claro e cinza médio
- Bordas: sutis, com transparência

Exemplo de sensação:
Quero que o usuário entre e pense:
“Agora eu finalmente entendi minha vida financeira.”

Não quero um app com cara de planilha.
Quero algo emocionalmente forte, visualmente limpo e com sensação de controle.

Crie o design para as seguintes telas:

1. Tela de login/cadastro
- Visual premium
- Frase de impacto
- Formulário simples
- Botão principal elegante
- Elementos visuais que remetam a clareza, controle e evolução financeira

2. Dashboard principal
Deve mostrar:
- Saudação ao usuário
- Resultado líquido do mês
- Receita mensal
- Despesas mensais
- Total em dívidas
- Previsão do fechamento do mês
- Cards com indicadores
- Gráfico de gastos por categoria
- Gráfico de evolução mensal
- Últimas transações
- Alertas inteligentes como:
  “Você gastou 27% a mais com alimentação este mês”
  “Se continuar nesse ritmo, seu mês fecha negativo”
  “Você tem R$ 450 livres até o fim do mês”

3. Tela de transações
- Lista de receitas e despesas
- Filtros por data, categoria, conta e tipo
- Botão para adicionar transação
- Visual claro para distinguir entrada e saída
- Estado vazio bonito quando não houver transações

4. Modal/Formulário de nova transação
Campos:
- Descrição
- Valor
- Tipo: receita, despesa, transferência, dívida, investimento
- Conta
- Categoria
- Data
- Status
- Observações

5. Tela de contas financeiras
- Cards para contas bancárias, dinheiro físico, investimentos e cartões
- Mostrar saldo atual
- Mostrar instituição
- Mostrar tipo da conta
- Botão para criar nova conta

6. Tela de dívidas
- Mostrar total em dívida
- Dívidas por credor
- Valor original
- Valor atual
- Status: aberta, renegociada, paga, vencida
- Prioridade
- Sugestão visual de qual dívida atacar primeiro
- Gráfico ou barra de progresso de quitação

7. Tela de orçamento mensal
- Categorias com valor planejado vs realizado
- Barras de progresso
- Destaques quando passou do orçamento
- Sugestões visuais para ajustar gastos

8. Tela de metas
- Metas como “sair do negativo”, “reserva de emergência”, “quitar cartão”
- Valor atual vs valor alvo
- Prazo
- Progresso visual
- Mensagem motivacional

9. Tela de importação CSV
- Área de upload bonita
- Etapas do processo:
  1. Enviar arquivo
  2. Mapear colunas
  3. Revisar transações
  4. Confirmar importação
- Estado de erro e sucesso
- Visual simples, sem parecer técnico demais

10. Tela de insights financeiros
- Cards com análises automáticas
- Separação por severidade:
  - Informação
  - Atenção
  - Perigo
  - Vitória
- Linguagem humana, não robótica
- Exemplos:
  “Seu gasto com delivery virou uma assinatura invisível”
  “Você está pagando dívidas, mas ainda sem estratégia”
  “Sua margem de sobra melhorou 12% este mês”

11. Configurações
- Dados do perfil
- Moeda
- Preferências
- Tema claro/escuro
- Gestão de categorias

Componentes necessários:
- Sidebar
- Header
- Cards financeiros
- Tabelas modernas
- Badges de status
- Botões principais e secundários
- Inputs
- Selects
- Modais
- Empty states
- Toasts
- Loading skeletons
- Gráficos
- Barras de progresso
- Cards de insight

Direção de UX:
- O app precisa ser simples para quem está perdido financeiramente
- Não pode parecer contabilidade difícil
- Use linguagem clara e emocional
- Mostre o problema, mas também a solução
- O usuário precisa sentir controle, não culpa
- Toda tela deve responder:
  “O que está acontecendo?”
  “Por que isso importa?”
  “O que eu faço agora?”

Direção visual:
- Cards com bordas arredondadas
- Sombras suaves
- Espaçamento generoso
- Ícones modernos
- Hierarquia visual forte
- Dados financeiros bem destacados
- Gráficos limpos
- Microinterações suaves
- Layout responsivo para desktop e mobile

Entregáveis:
1. Descreva o conceito visual do produto.
2. Defina a paleta de cores com HEX.
3. Defina tipografia.
4. Defina estilo dos componentes.
5. Crie a estrutura das telas.
6. Descreva o layout de cada tela.
7. Sugira microcopy para botões, cards e mensagens.
8. Sugira empty states.
9. Sugira animações e microinterações.
10. Gere um guia visual para um desenvolvedor implementar em React + TailwindCSS.

Importante:
Não quero um design genérico.
Quero algo com identidade própria, premium, emocional e muito claro.
Pense como um product designer sênior criando o primeiro design system de uma fintech pessoal.

Tipografia
Headings / Valores Financeiros: Georgia, serif — elegância clássica sem parecer banco velho. Usado nos valores em destaque, greeting, títulos de tela e números grandes de KPI.
Corpo / UI: -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif — leitura confortável, hierarquia limpa para labels, categorias e dados.
Escala: 36px (hero dívida) → 22px (KPI) → 20px (greeting) → 17px (card title) → 14px (body) → 11px (label/badge) → 10px (uppercase caps)
Componentes e Microcopy
Botão primário: background: #C9A84C, color: #060A14, font-weight: 700 — dourado sólido, texto navy escuro. Texto: + Transação, + Nova Meta, Editar Orçamento
Botão ghost: borda rgba(255,255,255,0.05), cor #8B95A8. Texto: Importar CSV, Ver detalhes
Badges: pill com border-radius: 20px, background 14% de opacidade da cor semântica. Texto em caixa alta 9.5px com letter-spacing: 0.05em
Cards de KPI: background: #141C35, borda dourada 14% opacidade, border-radius: 11px, label em caixa alta + valor em Georgia
Barras de progresso: height: 6–9px, fundo #0B1020, fill com cor semântica. Para "estourado", usar vermelho; para "ok", verde; para "atenção", dourado
Empty States sugeridos

Transações vazias: "Nenhuma transação ainda. Seu primeiro lançamento é o começo da clareza." + botão + Adicionar agora
Insights sem dados: "Ainda coletando dados. Quanto mais você lançar, mais inteligentes ficam os insights."
Metas sem cadastro: "Toda jornada começa com uma meta. Qual é a sua?"

Animações e Microinterações
Todas as animações devem ser ease-out, nunca linear. Durações: entradas 0.18s, barras de progresso 0.5–0.7s (com transition-delay por linha), toasts 0.2s entrada + 3s pausa + 0.2s saída.

Trocar de tela: opacity 0 → 1 + translateY(8px) → 0 no .screen.active
Hover em cards de conta: border-color sobe de 14% → 28% de opacidade dourada
KPI: counter animation em JS (0 → valor final) com requestAnimationFrame nos primeiros 600ms
Barras de progresso: width: 0 → X% no mount, com transition: width 0.6s cubic-bezier(0.34, 1.56, 0.64, 1)

Guia React + TailwindCSS
A estrutura base usa flex h-screen bg-[#060A14] text-[#F0EDE6]. A sidebar é w-52 shrink-0 bg-[#0F1628] border-r border-[#C9A84C]/14. Cards usam bg-[#141C35] border border-[#C9A84C]/14 rounded-xl p-5. Para as classes de cor semântica, crie um sistema de tokens via tailwind.config.js com os valores acima em extend.colors. Use font-['Georgia'] para todos os valores financeiros grandes e font-sans para UI.
